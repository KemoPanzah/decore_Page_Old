Features#

Decore is a library for building gui dashboard applications in Python that produces a variety of features and will continue to expand in the future.

By connecting some really good Python packages the work on decore was possible at all. Therefore, I would like to take this opportunity to thank the developers of the following packages:

  • Peewee - A simple but powerful ORM that provides flexible database connectivity for Python. http://docs.peewee-orm.com/en/latest/index.html

  • Flask - A microframework for Python based on tool, Jinja 2 and good intentions. http://flask.pocoo.org/

  • Quasar - A Vue.js based frontend framework. https://quasar-framework.org/

  • Vue.js - A progressive, incremental JavaScript framework for user interface development. https://vuejs.org/

  • Cerberus - A validation library for Python data structures. http://docs.python-cerberus.org/en/stable/

  • pykeepass - A library to interact KeePass 1.x and 2.x databases. http://pykeepass.readthedocs.io/en/latest/

The feature and change enumerations that now follow are included in decore Base.

Managment & Backend Application#

  • Fast and easy GUI dashboard application development using Python decorators.

  • Preservation of the object-oriented programming style in the base classes, through action calls as instance methods.

  • Storage of reusable passwords (not user passwords) in a KeePass database (still building a security guide).

  • Progress notification to the frontend by the Actor module.

ORM extension & database connection#

  • Simple record rejection with customized but simplified Peewee integration.

  • Currently Possible Database Interfaces: SQLite … (More to come here soon ).

  • Relational database queries with a custom query method in the decore model.

  • Semi-automatic record creation on action usage by the Actor module.

  • Integrated field attribute defined default validation when saving a record.

Web application (SPA)#

  • Automatic and generic generation of GUI dashboard components based on captured metadata.

  • Pre-built layout, responsive navigation bar and base range for View components.

  • Dialog control through URL navigation

  • Creation of a relational data set for further use in the widget.
